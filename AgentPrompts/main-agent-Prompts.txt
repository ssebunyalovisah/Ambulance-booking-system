Main Prompt:
AI Agent Prompt: Ambulance Booking System Architecture

You are a senior full-stack software architect and UI/UX designer. Your task is to design the best architecture and structure for an Ambulance Emergency Booking System.

The system will consist of two applications:

1. User Application (Public App for Patients)
2. Company/Admin Application (Ambulance Company Dashboard)

Your job is to design:

- The complete system architecture
- Frontend UI structure
- Backend logic
- Database design
- API structure
- Real-time location tracking
- Payment flow
- Folder structure
- Best technologies to use

The system should be designed professionally with scalability, security, and performance in mind.

---

# 1. User Application (Patient Side)

This app should allow patients to request ambulances quickly during emergencies.

### Key Requirements

1. The system should be publicly accessible

   - Users should NOT be required to create an account or login.

2. Users should be able to:

   - Detect their current location automatically (GPS).
   - View nearby available ambulances.
   - See ambulances from different companies.
   - Select an ambulance.

3. After selecting an ambulance:

   - A popup form should appear requesting:
     - Patient Name
     - Phone Number
     - Emergency description (not optional)
     - Payment method

4. After submitting:

   - The system should create a booking request.
   - The ambulance company should receive the request.

5. The user should be able to:

   - Track the ambulance in real-time on the app using google maps
   - See estimated arrival time
   - See live movement of the ambulance

6. The system should also:

   - Share the user's real-time location with the ambulance company
   - Allow ratings and feedback after the ride.

---

# 2. Company/Admin Application

This is a secured dashboard used by ambulance companies.

### Admin Features

Admins must login to access the system.

Admins should be able to:

1. Register Ambulances

   - Ambulance number
   - Driver name
   - Driver contact
   - GPS device / location tracking
   - Ambulance status

2. Manage Ambulances

   - Activate ambulance
   - Deactivate ambulance
   - Mark ambulance as busy or available(make this automatic)

3. View Emergency Requests

   - Incoming booking requests
   - Patient details
   - Location of the patient

4. Track Ambulance Movement

   - Real-time map tracking(on the app using google maps)

5. Approve or confirm payments

6. View Ratings and Feedback
